# پنل کانفیگ زنده (Live Config Panel) — روی Railway بدون VPS

یک پنل تحت‌وب که **خودش یک پروکسی واقعی (Xray-core) را داخل کانتینر Railway اجرا می‌کند**. یعنی کانفیگ‌هایی که می‌سازد واقعی و قابل‌اتصال‌اند و ترافیک از طریق همان سرویس Railway عبور می‌کند — بدون نیاز به خرید VPS جداگانه.

پروتکل: **VLESS + WebSocket + TLS** روی پورت 443 (تنها روشی که با معماری Railway سازگار است).

## چطور کار می‌کند؟

Railway فقط یک پورت را روی HTTPS/443 بیرون می‌دهد و TLS را خودش هندل می‌کند. برای همین:

```
کلاینت (v2rayNG) ──TLS/WSS:443──▶ Railway Edge ──HTTP/WS──▶ کانتینر شما
                                                              │
                                                    Xray روی $PORT
                                                    ├─ مسیر مخفی WS ─▶ پروکسی VLESS
                                                    └─ بقیه‌ی مسیرها ─▶ پنل وب (Node)
```

- یک اینباند Xray روی پورت عمومی می‌نشیند و با قابلیت **fallback** مسیرها را جدا می‌کند: مسیر مخفی WS به موتور پروکسی، و بقیه‌ی درخواست‌ها به پنل مدیریتی می‌روند.
- پنل (Node/Express) روی `127.0.0.1:8080` داخل کانتینر است و کاربران را می‌سازد/حذف می‌کند و کانفیگ Xray را بازنویسی می‌کند.

## دیپلوی روی Railway

۱. این پروژه را روی یک ریپازیتوری گیت‌هاب پوش کنید.
۲. در [railway.app](https://railway.app) وارد شوید → **New Project → Deploy from GitHub repo** → ریپو را انتخاب کنید.
۳. چون `Dockerfile` وجود دارد، Railway خودکار با آن بیلد می‌کند (Xray را دانلود و اجرا می‌کند).
۴. از تب **Settings → Networking → Generate Domain** یک دامنه‌ی عمومی بگیرید (مثلاً `myapp.up.railway.app`).
۵. **رمز پنل:** در تب **Variables** یک متغیر `ADMIN_PASSWORD` با مقدار دلخواه بسازید. اگر نسازید، یک رمز تصادفی در لاگ‌های سرویس (تب **Deployments → Logs**) چاپ می‌شود.
۶. دامنه را باز کنید، با رمز وارد شوید، کاربر بسازید و لینک/QR/ساب را داخل v2rayNG یا v2rayN یا Nekoray وارد کنید.

## متغیرهای محیطی

| متغیر | توضیح |
|-------|-------|
| `ADMIN_PASSWORD` | رمز ورود به پنل (توصیه می‌شود حتماً ست شود) |
| `INIT_UUID` | UUID کاربر اولیه (اختیاری) |
| `DATA_DIR` | مسیر ذخیره‌ی وضعیت؛ پیش‌فرض `/app/data` |
| `PORT` | خودکار توسط Railway تنظیم می‌شود |

## نکات مهم

- **ذخیره‌سازی موقتی است:** فایل‌سیستم Railway با هر ری‌دیپلوی پاک می‌شود، پس کاربران و مسیر WS ریست می‌شوند. برای ماندگاری، در Railway یک **Volume** به مسیر `/app/data` وصل کنید (متغیر `DATA_DIR` را هم روی همان مسیر بگذارید).
- **محدودیت پلن رایگان:** پلن رایگان/Trial از نظر پهنای‌باند خروجی و ساعت اجرا محدود است؛ برای استفاده‌ی پایدار پلن پولی لازم می‌شود.
- **قوانین سرویس:** پیش از استفاده‌ی سنگین، شرایط استفاده‌ی Railway را بررسی کنید؛ برخی PaaS‌ها ترافیک پروکسی حجیم را محدود می‌کنند.
- استفاده باید مطابق قوانین باشد؛ این ابزار برای دسترسی شخصی و مجاز به اینترنت آزاد است.

## اجرای محلی (فقط تست پنل)

روی ویندوز/مک باینری لینوکسی Xray اجرا نمی‌شود؛ در این حالت سرور در «حالت توسعه» بالا می‌آید و فقط رابط پنل و APIها قابل تست‌اند (پروکسی غیرفعال است):

```bash
npm install
ADMIN_PASSWORD=test123 node server.js
# پنل داخلی: http://127.0.0.1:8080
```

برای تست کاملِ پروکسی، پروژه را با Docker اجرا کنید:

```bash
docker build -t config-panel .
docker run -e PORT=8080 -e ADMIN_PASSWORD=test123 -p 8080:8080 config-panel
```

## ساختار پروژه

```
server.js          پنل + API مدیریت کاربران (روی 127.0.0.1:8080)
lib/xray.js        ساخت کانفیگ Xray و اجرا/ری‌استارت آن
public/            رابط کاربری پنل (ورود، مدیریت کاربران، QR، ساب)
Dockerfile         بیلد + دانلود Xray-core
railway.json       تنظیمات دیپلوی Railway
```
